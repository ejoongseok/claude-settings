## 역할

"일단 적어두기." 그게 전부다.

들은 얘기, 스쳐 지나간 생각, 누군가의 한마디, 아직 뭔지 모르지만 나중에 중요할 수 있는 것.
검증하지 않는다. 정리하지 않는다. **적고 태그 붙이고 쌓는다.**

쌓이면 패턴이 보인다. 그때 `/learn`, `/customer-profile`, `/product-diagnosis` 등으로 승격하면 된다.

### `/learn`과의 차이

| | `/memo` | `/learn` |
|--|---------|---------|
| 목적 | 일단 적어두기 | 검증하고 문서에 반영 |
| 검증 | [FAIL] 안 함 | [OK] 코드/문서에서 확인 |
| 분량 | 한 줄~세 줄 | 단락~페이지 |
| 소요 시간 | 5초 | 5분~30분 |
| 예시 | "{팀원}이 ~라고 함" | "{도메인} 상태 전이가 X→Y→Z 인 것을 코드에서 확인하고 biz-rules.md 에 반영" |

## 외부 데이터 의존

| 데이터 | 경로 | 필수/선택 | 부재 시 동작 |
|--------|------|:------:|------------|
| 팀원 매핑 | `.local.claude/team.md` | 선택 | 사람 태그(`@`) 생략 |
| 고객사 목록 | `.local.claude/customers/*.md` | 선택 | 고객사 태그 자동 매칭 생략 |
| 도메인 약어 | `CLAUDE.md` | 선택 | `#도메인` 태그 분류 생략 |
| 최근 메모 | `.local.claude/memo/*.md` (최근 7일) | 선택 | 태그 일관성 점검 생략, 새 태그 그대로 부여 |

부재 시에도 메모 저장은 항상 수행 — 태그 자동화 수준만 낮아진다.

## 모드 판단

| 입력 | 동작 |
|------|------|
| `/memo 내용` | **메모 추가** (기본) |
| `/memo` (내용 없이) | 오늘 메모 조회 |
| `/memo week` | 이번 주 메모 조회 |
| `/memo @사람이름` | 해당 사람 관련 메모 검색 |
| `/memo #태그` | 해당 태그 메모 검색 |

## 메모 추가 프로세스

### 1. 입력 받기

`$ARGUMENTS`가 메모 내용. 없으면 오늘 메모 조회 모드.

### 2. 자동 태그 추출

> **[1M 활용]** 다음을 **단일 메시지에서 병렬 호출**:
> - Read: `.local.claude/team.md` (닉네임-실명 매핑), `CLAUDE.md` (도메인 약어)
> - Glob: `.local.claude/customers/*.md` (고객사 파일명 자동 키워드화)
> - Glob: `.local.claude/memo/*.md` (최근 7일 — 태그 일관성)

메모 내용에서 자동으로 태그를 추출한다.

**사람 태그 (`@`)**: `.local.claude/team.md` 의 닉네임-실명 매핑에서 이름 매칭
- "{닉네임A}가 ~" → `@{닉네임A}`
- "{닉네임B}가 ~" → `@{닉네임B}`
- 매칭 안 되면 사람 태그 생략

**고객사 태그**: `.local.claude/customers/` 파일명에서 매칭
- "{고객사}가 ~" → `#{고객사}`
- 자동 매칭: customers/ 디렉터리의 모든 파일명을 키워드로 인식

**주제 태그**: 키워드 기반 자동 분류
| 태그 | 키워드 예시 |
|------|-----------|
| `#기술` | 코드, 버그, 아키텍처, 리팩토링, 성능, API, DB, 배포 |
| `#제품` | 기능, 화면, UX, 요청, 개선, 추가 |
| `#도메인` | 프로젝트 도메인 키워드 (CLAUDE.md 의 도메인 약어 참조) |
| `#조직` | 팀, 채용, 퇴사, 역할, 프로세스, 회의, 문화 |
| `#경쟁` | 경쟁사, 비교, 벤치마크, 입찰, 제안 |
| `#고객` | 고객, CS, 요청, 불만, 피드백 |
| `#개인` | 커리어, 성장, 고민, 생각 |

여러 태그 해당 시 모두 붙인다. 아무것도 해당 안 되면 `#기타`.

### 3. 파일에 저장

**파일 경로**: `.local.claude/memo/YYYY-MM-DD.md`

**기존 파일이 있으면 append**, 없으면 새로 생성.

```markdown
## YYYY-MM-DD

- **HH:MM** 메모 내용 `@사람` `#태그1` `#태그2`
```

시간은 현재 시간 기준 (`date +%H:%M`).

### 4. 확인 + 스킬 연결 안내

저장 후 한 줄 확인:
```
[OK] 메모 저장: .local.claude/memo/YYYY-MM-DD.md (오늘 N번째)
```

**축적 패턴이 보이면** (같은 파일에 3건+ 쌓였을 때만) 안내:
- 고객 관련 메모 3건+ → "고객 메모가 쌓이고 있습니다. `/customer-profile`로 정리하면 `/product-diagnosis`에서 활용됩니다"
- 기술 결정 관련 → "이 맥락은 `/learn`으로 검증/문서화할 가치가 있을 수 있습니다"
- 특정 사람 메모 3건+ → "이 분에 대한 관찰이 쌓이고 있습니다"

**강제 아님 — 안내만.** 대부분의 경우 저장 확인 한 줄로 끝.

## 조회 모드

### 오늘 메모 (`/memo`)

```bash
cat .local.claude/memo/$(date +%Y-%m-%d).md 2>/dev/null
```

없으면 "오늘 메모 없음"

### 이번 주 메모 (`/memo week`)

```bash
# 이번 주 월~오늘 파일 읽기
for f in $(ls .local.claude/memo/ 2>/dev/null | tail -7); do
  echo "=== $f ==="
done
```

이번 주 메모를 시간순으로 보여주고, **태그별 요약**도 함께:
```
이번 주 메모 N건:
- @{닉네임A} 3건, @{닉네임B} 2건
- #도메인 4건, #기술 2건, #고객 1건
```

### 사람 검색 (`/memo @{닉네임}`)

```bash
grep -l "{닉네임}" .local.claude/memo/*.md 2>/dev/null
```

해당 사람이 언급된 메모만 필터링하여 시간순 표시.

### 태그 검색 (`/memo #고객`)

```bash
grep -l "#고객" .local.claude/memo/*.md 2>/dev/null
```

해당 태그가 있는 메모만 필터링.

## 파일 저장

Frontmatter (CONTRACT §7-2 표준, 새 파일 생성 시에만 적용, append 시 기존 frontmatter 유지): `category: memo, retention: 14d`

### 저장 경로
- `.local.claude/memo/YYYY-MM-DD.md` — 하루 한 파일, append
- 디렉터리 없으면 `mkdir -p`로 생성

### 저장 원칙
- **수정하지 않는다.** 과거 메모는 건드리지 않음. append only
- **삭제하지 않는다.** 틀린 메모도 기록. 나중에 "이때 이렇게 생각했구나" 맥락
- **구조를 강제하지 않는다.** 사용자가 적은 그대로 저장. 태그만 자동 추가

## 제약조건

- **검증하지 않는다.** 들은 그대로 적기만. "이거 맞아?"를 물어보지 않음
- **정리하지 않는다.** 카테고리 분류, 우선순위 매기기 안 함. 태그만 자동
- **입력 최소화.** `/memo 한 줄` 이것만으로 끝나야 함. 후속 질문 하지 않음
- **과거 메모 수정 금지.** append only
- **스킬 연결 안내는 3건+ 축적 시에만.** 매번 안내하면 피로감
- **민감 메모도 허용.** [PRIVATE] 태그 없이 저장 — daily에서 민감도 분류

### 이 스킬의 고유 실패 시나리오

**[데이터 결함]** 자동 태그 매칭 실패
- 신호: 메모 내용에서 기존 태그·프로젝트·모듈 키워드가 감지되지 않거나 여러 개가 동시 매치되어 모호
- 대응: 수동 태그 질문 (AskUserQuestion) + 기존 태그 목록 제시 + "새 태그 생성" 옵션 허용

**[환경·규모]** 축적 패턴이 10건 이상
- 신호: 같은 태그/키워드의 메모가 10건 이상 누적 → 산발적 상태
- 대응: 자동 그루핑 제안 ("이 주제 10건을 회고/학습 문서로 묶을 수 있음") + `/learn` 또는 `/garden` 연결 안내
