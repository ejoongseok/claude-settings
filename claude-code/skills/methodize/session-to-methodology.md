# 세션에서 방법론 추출, AI 페어링 대화를 재사용 자산으로

> **무엇**: AI(Claude 등)와의 긴 페어링 세션에서 "이 방식, 또 쓰겠다" 싶은 순간이 오면, 그 작업 방식(순서와 판단 기준)을 **재사용 방법론 문서**로 추출하고, 방식이 만들어진 **대화 원문과 발췌를 학습 자료**로 남긴다.
>
> **왜**: 예전에는 페어 프로그래밍으로 "일하는 방식"을 어깨너머로 전수했다. AI 페어링은 그 과정이 전부 텍스트로 남으므로, 다듬으면 **동료와 신입이 비동기로 읽는 교재**가 된다. "이 상황에서 이렇게 개입하니 이런 결과가 나왔다"처럼.
>
> **구조**: 원문이 단일 원본(SSoT, 모든 파생물의 기준)이고, 방법론과 발췌는 원문에서 언제든 다시 만들 수 있는 파생물. 깔때기처럼 증류한다: 원문[원석]에서 여정, 구조, 방법론과 발췌[알맹이]로.

용어. **컨텍스트 압축(compaction)**은 긴 대화에서 오래된 내용이 요약으로 대체되는 것(원문 표현 소실). **세션 기록(transcript)**은 도구가 세션마다 디스크에 남기는 대화 기록 파일(압축과 무관하게 원문 보존, 보관 기한 있음). **ledger**는 어느 세션에서 무엇을 추출했는지 적는 처리 대장.

## 언제 쓰나 (사용 시나리오)

긴 세션(large-context) 전제. 호출 시점 3가지:

| 시점 | 상황 | 동작 |
|------|------|------|
| **체크포인트 (1차)** | 세션 중 "방법론감" 순간(품질 좋은 결과물이나 오래 주고받은 산물), **압축 전** | 원문 확보와 추출. 압축으로 사라지기 전에 확보하는 것이 핵심 |
| **재호출** | 압축 후 대화가 이어져 끝났을 때 | **같은 문서로 증분 수렴**. 문서 2개 금지. 델타만 처리 |
| **사후 회수** | 세션 종료 후 (기록 보관 기한 내) | 세션 기록에서 원문 회수 후 추출 |

## 추출 대상의 구분 (무엇이 "방법론"인가)

재사용 단위가 **절차(어떻게)**면 이 방법론의 대상이다. **사실(무엇)**이면 지식 노트로, **단건 교훈**이면 트러블슈팅 노트로 보낸다. 반복 구조가 보일 때 방법론으로 승격한다.

## Part 1. 추출 파이프라인 (10단계)

각 단계는 추출자의 행동이고, 앞 산출이 다음 입력. 아래 표의 순서대로 흐른다.

| 순서 | 단계 | 무엇 | 게이트 |
|:---:|------|------|:---:|
| 1 | 트리거와 가치와 매칭 | "방법론감" 판단 + 가치 3질문 + 기존 방법론 매칭(신규, 증분, 추출 안 함) | **가치 게이트** |
| 2 | **원문 확보 [원석]** | 대화 전체를 세션 기록에서 기계 추출해 영구 파일로. 이후 모든 인용의 유일 출처 | 원문 우선 |
| 3 | 여정 정리 [1차 증류] | 원문 기반 타임라인 (재호출 시 델타만) | - |
| 4 | 구조 인식 [2차 증류] | 지배 구조 판별 (혼합 허용) | - |
| 5 | 층위 분리 | 범용 후보 대 도메인 특화 | - |
| 6 | 기억 보강 | 원문에 없는 암묵지를 사용자에게서 끌어냄 (4기법) | **검증 게이트** |
| 7 | 정합성 다듬기 | 중복, 모순, 용어 재독 | - |
| 8 | 회고 | 비효율과 왕복(못한 것)까지 | - |
| 9 | 본체 반영 | 회고 교훈을 방법론에 흡수 | - |
| 10 | 승격과 증분 | 신규 문서화 또는 기존 증분(승인)과 전파 제안 | **자체 비판 게이트** |

### 가치 판정 (기본값: 추출 안 함)

3질문에 답을 **표기**하고 판정한다 (AI 단독 판정은 관대해진다. 근거 표기와 사용자 확정이 견제 장치):

1. 반복될 작업인가? (1회성이면 추출 안 함)
2. 남에게 전파할 가치가 있나?
3. 기존 방법론의 사례 1건으로 충분한가? (충분하면 증분이나 사례 추가로)

추출 안 함 판정도 ledger에 남긴다(재호출 시 재판정 방지). 매칭은 방법론 표식(frontmatter `category: methodology`)을 지식베이스 **전역**에서 grep한다. 방법론 폴더 밖에 있는 문서도 잡는다(스캔 사각이 중복 문서를 만든다).

### 원문 확보 (원문은 추출의 단일 원본)

- **소스**: 세션 기록 파일. 압축과 무관하게 원문이 실시간 기록된다. "휘발하는 건 대화 창의 컨텍스트이지 디스크 기록이 아니다"
- **추출**: 사용자와 AI 대화 텍스트만 (도구 호출과 시스템 노이즈 제외)
- **함정 (실측)**: 사용자 발화는 저장 경로가 하나가 아니다. 일반 메시지 외에 **도구 거부 사유와 질문 답변(tool_result 내장), 작업 중 도착 메시지(별도 타입)**가 있고, 기본 필터는 이들을 놓친다. 표적 회수를 추가하고, **추출 후 핵심 발화가 원문에 실재하는지 grep으로 검증**한다
- **원칙**: 이후 모든 따옴표 인용은 이 파일에서만. **기억으로 인용을 재구성하지 않는다**. 압축 후 기억 기반 인용은 사실상 날조다

입력 소스 폴백 (세션 기록 접근 실패 시): 라이브 대화(압축 전 구간만 신뢰), 산출물 변경 이력, 사용자 재진술(`[재구성]` 태그 강제, 따옴표 금지).

### 구조 인식과 층위 분리

- 구조 인식 판별 질문: 앞 산출이 다음 입력이면 **파이프라인**, 순서 무관 점검이면 **체크리스트**, 판단 규범이면 **원칙**. 실제 방법론은 대부분 혼합이라 지배 구조 하나를 고르고 나머지는 보조로
- 층위 분리의 "범용" 라벨: 명시 적용 2건 전까지 **"범용 후보"**로만 (1건에서 "어디에나 적용" 단정 금지)

### 기억 보강 (4기법과 검증 게이트)

AI는 원문에 있는 것만 안다. 사용자만 아는 암묵 단계가 여기서 들어온다. 능동 4기법:

1. **시드 체크리스트 질문**. 작업 유형별 표준 요소(부록 A)를 제시하고 "이것도 하셨나요?"
2. **여정 빈칸 탐지**. 원문에 명시된 것과 했을 법한데 안 드러난 것을 구분해 후자를 질문
3. **"이것 말고 또?" 반복**. 2회 연속 새 발견이 없으면 종료
4. **암묵지 신호 포착**. "~한 것 같은데", "분명 했을", "뭐 등등"이 파고들 신호

**검증 게이트 (작화 방지)**: "분명 했을"은 암묵지 신호이자 **작화(confabulation, 사실이 아닌 것을 사실처럼 회상) 신호**다. 기억으로 복원된 단계는 산출물 흔적(파일, 커밋, 문서)과 대조한다. 흔적 확인이면 `[실측]`, 흔적 없음이면 `[기억-미확인]`(본문이 아닌 후보란에 격리).

### 승격과 증분 (자체 비판 게이트)

- 승격 직전 **자체 비판 재독 필수**. "정직성 원칙을 이 문서 자신에게 적용했는가"(사례 수 표기, 한계, 회고)
- 공개 예정 방법론은 **적대적 검토**(red-team 계열) 권장
- 산출 형식: 결론 먼저, 훑어도 구조가 잡히게(표와 라벨 불릿), 독자 우선. 외부 노출본은 내부 경로 금지, 용어 풀어쓰기, 정량 표기 추가

### 다중 세션과 체크포인트 누적 (안전장치)

방법론은 살아있는 문서다. 세션과 체크포인트가 쌓이며 성장한다. 단 안전장치:

- **append**. 새 사례를 부록 표 1행으로 (상세는 발췌 파일 링크로, 본체 비대 방지, 임계 약 250줄)
- **보강**. 새 통찰로 보완, `(출처: 세션 ID와 날짜)` 표기
- **수정**. 반증 시 정정과 **이력 보존** (지운 흔적 없이 정정 표기)
- **CONFLICT**. 기존 서술과 모순이면 **자동 수정 금지**하고 좌우 비교표와 사용자 확정
- **증분 승인**. 기존 문서 수정은 diff 미리보기와 승인 후
- **은퇴와 병합**. 매칭에서 주제 중복 발견 시 병합 분기, 장기 무적용 시 아카이브 이동

## Part 2. 학습용 발췌 (원문과 발췌 이원화)

방법론(결과)만 남기면 "어떻게 거기 도달했나"가 사라진다. 대화 궤적이 그 "어떻게"다. 원본 보존과 해석 분리 패턴:

### 원문 (raw)

- 경로: `methodology/cases/raw/{세션ID}.md`
- frontmatter: `type: session-raw`, 세션ID, 체크포인트 수, `retention: permanent`
- 본문: `## Checkpoint N ({시각})` 섹션 append-only. 세션 단위 1파일 (한 세션에서 방법론 2개가 나와도 원문은 공유)

### 발췌

- 경로: `methodology/cases/{방법론명}-{날짜}-{주제}.md`
- **발췌 단위는 4연쇄**: before 상태, 사용자 발화(원문 인용), AI 반응, 산출 변화. 발췌마다 맥락 1줄과 So What 1줄
- **독자**: 방법론을 배우려는 동료와 신입. 발췌만 읽어도 "왜 그 전환이 일어났나"가 보여야 한다
- **마스킹**: 시크릿과 개인정보 필수. 타인 발화 인용은 내부 공유도 주의 (공개는 별도 스크럽으로 고객사명, 인물, 계정, URL)

방법론 부록 사례 표에 발췌 링크 컬럼을 둔다. **적용만 한 세션도 1행 추가**(재사용 추적으로, 승격과 은퇴 판단의 데이터).

## Part 3. 전파 경로 (만들기만 하면 개인 폴더에서 낡는다)

개인 지식베이스는 대개 공유되지 않는다. 전파는 별도 행동이 필요하다:

| 경로 | 대상 | 수단 |
|------|------|------|
| **사내** | 팀 동료와 신입 | 사내 게시 채널(위키, 게시판, 메신저 문서)로 발행 |
| **외부** | 공개 (발표, 블로그, 저장소) | 중립화(익명화와 용어 풀기) 후 공개본 배포 |

파이프라인 마지막 단계인 승격에서 전파 제안이 내장된다 (실행은 사용자 판단).

공개본 규칙: 내부본이 단일 원본이다(공개본 frontmatter에 원본 경로와 마지막 동기를 기록해 어긋남 검출). 회사와 프로젝트와 인물 익명화. 구체 버전과 도구는 "예시" 표기. **정직 표기 필수**(`provenance: n=1`과 한계 명시). "실증된 패턴" **주장**의 자격은 명시 적용 2건 이상에 그중 1건이 타 도메인이다. 사후 부합(이미 있던 것을 나중에 패턴에 끼워 맞춤)은 사례로 세지 않는다.

## Part 4. 원칙과 이 문서의 회고

### 원칙

- **원문이 단일 원본**. 방법론과 발췌와 여정은 재구성 가능한 파생물. 대체 불가한 원문을 최우선 확보
- **사용자 기억이 핵심 입력**. 기억 보강 없으면 AI 단독 추출은 불완전하다. 단 기억은 검증 게이트를 거친다
- **회고(비효율) 필수**. 잘한 것만 담으면 미화
- **정직성 강제**. 한계와 미검증과 사례 수를 그대로 표기. 사례 아닌 것을 사례로 세지 않는다
- **사용자 참여 필수**. 무응답이면 `[미보강]`으로 남기고 진행, 완성 주장 금지

### 이 문서 자신의 회고 (초안에서 공개본 사이에 있었던 일)

초안은 다음을 범했다. 전부 이 문서가 스스로 세운 원칙 위반이었고, 적대적 검토(3회)에서 발견됐다:

- **사례 수 과장**: 병행 작성된 다른 방법론 문서를 "이 패턴의 2번째 적용이자 실증"으로 소급 인용했다. 실제로는 핵심 단계(기억 보강) 미수행, 대상 프로젝트 미완, 계보 미조사의 사후 끼워맞추기였고 부록 B로 정직화했다
- **게이트 0개**: 가치와 검증과 자체비판 게이트가 없었다. 적대 검토 자체가 "빠진 단계"였고, 사용자가 수동으로 시켜야 했다
- **입력 미정의**: "AI의 기억"이 유일한 입력이라 압축이나 과거 세션에서 붕괴했고, 원문 우선 구조로 재설계했다
- 교훈: **메타 방법론도 자기 자신에게 적용해야 한다** (자기 회고, 자기 게이트, 자기 정직성)

## 향후. 코퍼스 분석 (방법론이 여럿 쌓이면)

축적물(방법론 문서들, 원문과 발췌, ledger)은 방법론 모음집을 넘어 **"내가 AI와 일하는 방식"의 데이터셋**이 된다. 충분히 쌓이면 횡단 분석이 가능해진다:

- **개입 시그니처 추출**. 발췌에 쌓인 사용자 발화를 횡단 분석하면, 반복되는 효과적 개입(예를 들어 완전성 압박, 최악 케이스 대입 지시, 적대 검토 요청)이 보인다. 이것이 남에게 전수 가능한 "코칭 무브" 목록
- **타이밍 패턴 교정**. 늘 사용자가 시켜야 일어나던 개입을 파이프라인 게이트로 선제화 (이 방법론 자체가 첫 사례다. 회고와 적대 검토가 매번 사용자 트리거였고, 기억 보강과 승격 게이트로 내장했다)
- **공통 원칙 승격**. 여러 방법론에 반복 등장하는 원칙을 상위 공통 원칙집으로
- **사용 습관 진단**. ledger로 체크포인트 타이밍(압축 전에 잘 부르는가), 기각률, 추출한 방법론의 실제 재사용률

단, 문서 2, 3개 시점에 만들면 과잉 설계다. 시작 조건 예로는 방법론 5개 이상 또는 발췌 사용자 발화 30건 이상.

### 사례 1 궤적 (요약)

가치 판정에서 사용자가 "이 조사 방식, 다음에도 쓸 수 있겠다, 추출 가능한가?" 질문. 이어 여정을 되짚었다. 구조 인식은 사용자의 "파이프라인처럼 이어진다" 지적으로 확정됐다. 층위 분리로 조사 원칙(범용 후보)과 도입 검토(특화)를 갈랐다. 기억 보강에서 사용자가 놓친 요소(트레이드오프와 케이스 정리)를 회상으로 보강했고, 이것이 능동 4기법의 필요 근거가 됐다. 회고는 사용자 트리거로 비효율을 짚었다. 본체 반영으로 교훈을 흡수했다(착수 준비 단계 신설). 마지막으로 방법론 문서로 승격했다.

주: 이 궤적에 원문 확보 단계는 없었다. 그 결과가 "압축 후 기억 기반 인용은 날조 위험"으로 드러났고, 원문 확보가 신설됐다. **초안의 왕복 자체가 이 방법론의 개선 입력이었다.**
