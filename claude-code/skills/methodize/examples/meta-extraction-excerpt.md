# 발췌 예시, 방법론은 이렇게 태어난다

> **배경 (이것만 알면 됩니다)**: 작성자가 AI(Claude)와 "기술 도입 타당성 검토" 작업을 마친 뒤, 그 조사 방식을 재사용 문서로 만드는 대화를 나눴다. 아래는 그 대화에서 **사용자의 개입 한 마디가 방법론의 단계 하나씩을 만들어 간 순간들**의 원문 발췌다. 발췌 단위는 before(개입 전 상태), 사용자 발화(원문), AI 반응, 산출 변화.

## 발췌 1. 구조 인식: "파이프라인"이라는 한 마디

- **맥락**: AI가 조사 방식을 단계 목록으로 나열한 직후
- **before**: 방법론이 체크리스트(순서 무관 점검 목록)처럼 정리되고 있었음

> 뭔가 파이프라인 처럼 이어지잖아. (원문 발췌)

- **반응**: "앞 단계 출력이 다음 단계 입력"임을 인식하고 전체를 파이프라인으로 재구성
- **산출 변화**: 방법론 본체가 "질문 흐름"(각 단계는 조사자의 질문이고, 앞 답이 다음 입력)으로 바뀜
- **So What**: 구조 인식(파이프라인, 체크리스트, 원칙)이 문서의 뼈대를 결정한다

## 발췌 2. 기억 보강: "뭐, 등등"은 신호다

- **맥락**: AI가 추출을 다 했다고 여기던 시점. 트레이드오프와 케이스 정리가 빠진 줄도 몰랐음

> 그리고 케이스별로 나눠서 정리하고 트레이드 오프랑 뭐, 등등 했던거같기는한데 (원문 발췌)

- **반응**: 놓친 요소(트레이드오프, 케이스, 안전망, 의사결정 기록)를 별도 파트로 추가
- **산출 변화**: AI 단독 추출의 불완전함이 실증됐다. 기억 보강이 파이프라인의 핵심 단계로. "뭐, 등등"이 암묵지 신호로 등재
- **So What**: 사용자의 흐릿한 회상("~한 것 같은데")을 흘리지 말고 파고들라. 거기에 빠진 반쪽이 있다

## 발췌 3. 회고: "잘한 것만 담으면 미화"

- **맥락**: 방법론이 성공 패턴만으로 정리돼 가던 시점

> 그리고 지금 방식에서 분명 비효율적이거나 더 효과적으로 할 수 있었던 방법도 있었을거야. (원문 발췌)

- **반응**: 정정 반복, 얕은 병렬 조사, 늦은 최악케이스 대입 등 비효율을 별도 표로 회고
- **산출 변화**: 회고가 본체에 반영(착수 준비 단계 신설). "원칙은 맞았으나 타이밍이 늦었다"는 메타 교훈
- **So What**: 방법론에 실패 패턴이 없으면 다음 사용자가 같은 왕복을 반복한다

## 발췌 4. 메타 전환: 결과물보다 과정이 자산이다

- **맥락**: 1차 방법론이 완성되고 스킬화를 준비하던 시점. 재사용 자산은 그 문서 하나라고 여겼음

> 나는 오히려 이 대화를 추출하는 거를 방법론으로 만들고싶은데. 그다음에 스킬로 만들고 왜냐면. 요즘은 이런식으로도 지식을 전파해줄수가 있잖아. 문서로 만들어서. 예전에는 페어를 했다하면. 내가 클로드 코드랑 티키타카 하면서 한 플로우 자체가 다듬고 고도화하면 해당 케이스에서는 이렇게 할 수 있구나. 그리고 이런 결과가 나오는구나 이런거를 알려줄 수 있는거니까. 이게 더 가치있을거같지않나? (원문 발췌)

- **반응**: 추출 과정 자체를 9단계 메타 방법론으로 정식화하고 학습용 발췌(이 문서 같은) 개념 도입
- **산출 변화**: 이 방법론의 탄생. 페어링의 지식 전파를 비동기 문서로 대체하는 구조
- **So What**: 1차 산출물 뒤에는 종종 더 큰 메타 자산(그걸 만든 방법)이 숨어 있다

## 발췌 5. 사용 모델: 체크포인트와 수렴

- **맥락**: 스킬 설계 승인 직전. AI의 설계는 "세션 종료 후 1회 추출" 모델이었음

> 내가 이거를 만들려는 목적은 1M context다 보니까 대화를 오래한단말이지? 근데 세션을 진행하면서 이거는 방법론으로 남겨두면 비슷한 상황에서 써먹을 수 있겠다. 싶은 경우들이 있어. (…) 컴팩트되기 전이나 이럴때 한번씩 사용하려고하는거야. 근데 압축을 하고나서도 대화가 이어지면 끝나고 나서 한번더 해야하잖아. 근데 이게 스킬을 두번쓴다고 문서가 두개가 생기면 안되니까 그런거고. (원문 발췌, 중략 표시)

- **반응**: 처리 대장(ledger)의 의미를 "한 번 추출했으면 차단"에서 "같은 문서로 수렴"으로 뒤집음
- **산출 변화**: 압축 전 체크포인트가 1차 사용 모델로 확정됐다. 인용 날조의 1차 완화가 사용 패턴 자체가 됨
- **So What**: 도구 설계는 사용자의 실제 사용 장면에서 출발해야 한다. 설계자가 상상한 시나리오가 아니라

## 발췌 6. 깔때기: 원문이 단일 원본 (누락됐다 회수된 발화)

- **맥락**: 승인 직전 마지막 교정. **이 발화는 도구 거부 메시지에 담겨 있어 기본 추출이 놓쳤던 것**이다. 표적 회수로 확보됨(그 사건이 "사용자 발화 다중 경로" 규칙을 만들었다)

> 근데 이거 세션에서 맥락을 파악한다음에 전체 프롬프트를 추출하는게 1번이고 거기서 단계를 거치면서 뭔가 알맹이만 남겨야할거같기도한데 맞나? (원문 발췌, 보강 회수)

- **반응**: 파이프라인을 원문 확보 우선으로 재배열했다. 원석 먼저, 제련 다음
- **산출 변화**: 인용 날조가 "게이트로 방지"에서 "구조로 소멸"로 격상. 원문이 단일 원본, 나머지는 재증류 가능한 파생물
- **So What**: 대체 불가능한 것(원문)부터 확보하면 나머지는 언제든 다시 만들 수 있다

## 정리. 한 세션이 만든 파이프라인

| 사용자 개입 | 태어난 단계와 장치 |
|------|------|
| "파이프라인 처럼 이어지잖아" | 구조 인식 |
| "트레이드 오프랑 뭐, 등등 했던거같기는한데" | 기억 보강과 암묵지 신호 |
| "분명 비효율적이거나" | 회고 |
| "이 대화를 추출하는 거를 방법론으로" | 메타 방법론 자체 |
| "컴팩트되기 전이나 이럴때 한번씩" | 체크포인트와 수렴 모델 |
| "전체 프롬프트를 추출하는게 1번이고" | 원문 우선 깔때기 |
