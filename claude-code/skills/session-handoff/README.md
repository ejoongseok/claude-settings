# /session-handoff

세션이 끝나기 직전, **다음 세션의 모델이 곧바로 이어서 일할 수 있도록** 인계 문서를 한 장 만듭니다. 사람이 읽는 요약이 아니라, 다음 모델이 read 하고 즉시 재진입하도록 설계된 명령형 + 구조화 데이터 문서입니다.

## 왜 쓰나

긴 작업 세션의 컨텍스트는 세션이 닫히는 순간 사라집니다. 다음에 다시 시작하면 "내가 뭘 하고 있었지, 어디까지 했지, 뭘 정정했지"를 처음부터 복원해야 합니다. 그 복원 비용을 **사람에게서 인계 문서로** 옮깁니다. 다음 세션이 문서 한 장만 read 하면 바로 이어서 일합니다.

**컨텍스트 압축(`/compact`)이나 항구적 규칙(`CLAUDE.md`)과는 다릅니다.** 압축은 같은 세션을 잇고, `CLAUDE.md`는 늘 적용되는 규칙을 둡니다. 이 스킬은 그 사이, 즉 *한 작업의 끝*에서, 다음 세션 모델이 곧바로 재진입하도록 작업 유형, 정량 지표, 다음 액션, 미해결 가설을 구조화한 인계 문서를 남깁니다. 요약이 아니라 의도적 핸드오프입니다.

## 쓰면 얻는 것

- **다음 세션 즉시 재진입**: 무엇을 read 해야 하는지, 다음 액션이 뭔지, 어디서 시작하는지가 명령형으로 정리됩니다.
- **작업 유형 자동 식별**: sprint, pr, debug, doc, outreach, exploration, meta, retro 등 10종을 대화, git, 디렉터리 신호로 판별해, 유형에 맞는 인계 형태를 만듭니다.
- **참조 자산 자동 수집**: 본 대화에서 읽은 파일 + 사용자가 짚은 문서 + (선택) 메모리 인덱스를 3채널로 모아 우선순위로 정렬합니다.
- **정량 지표 보존**: "대략", "여러" 같은 정성 표현 대신 raw 수치(진척률, 변경 라인, 가설 N개 등). 추출 불가는 `[미측정]`으로 정직하게.
- **엣지케이스 방어**: 짧은 대화, git detached, 자산 50개 초과, 클립보드 실패 등 10종을 사전 점검하고 처리합니다.
- **민감 정보 redact**: 비밀번호, 토큰, 키, 개인정보 형태를 인계 문서에 옮기지 않습니다.

## 무엇을 얻으려는 건가 (목적)

예쁜 회고록이 아니라 **컨텍스트 연속성**입니다. 성공 기준은 "잘 정리됐다"가 아니라 **"다음 세션이 이 문서만으로 끊김 없이 이어서 일할 수 있는가"**입니다.

## 이럴 때

작업이 길어져 한 세션에 안 끝날 때, 큰 발견이나 정정 직후에 맥락을 남기고 싶을 때, 또는 컨텍스트가 가득 차 새 세션으로 넘어가야 할 때.

## 동작 방식 (요약)

1. 작업 유형 식별 + 본 대화 전체 흡수
2. 참조 자산 3채널 수집과 정렬 + 정량 지표 추출 + 발견, 정정, 결정 추출
3. 민감 정보 redact
4. 다음 세션 모델용 인계 문서 작성 (`.claude/handoff/`)
5. `LATEST.md` 갱신 + `INDEX.md` append
6. 사람용 콘솔 안내 + 재진입 프롬프트 클립보드 복사

다음 세션에서는 `@.claude/handoff/LATEST.md 읽고 진행해` 한 줄이면 됩니다.

## 결과물 예시

`examples/debug-order-api-500.md` 는 가상의 디버깅 세션을 인계한 예시입니다(내부 정보, 실제 자격증명 없음). 작업 유형 분기, 정량 지표, 발견과 정정, 민감 정보 redact, 다음 액션, stale 검증이 실제로 어떻게 채워지는지 한눈에 볼 수 있습니다. 양식의 기준은 `SKILL.md`의 템플릿이고, 이 예시는 결과물 형태를 미리 보기 위한 것입니다.

## 독자 분리

- **인계 문서**는 다음 세션 모델용: 명령형 지시 + frontmatter 메타 + 표와 구조화 데이터.
- **콘솔 출력**은 사람용: 한국어 친화 안내 + 입력할 명령어.

## 설치

이 폴더를 `~/.claude/skills/` 또는 프로젝트 `.claude/skills/` 아래에 그대로 두면 됩니다. 외부 의존이 없습니다. 인계 문서는 작업 폴더의 `.claude/handoff/` 아래에 쌓입니다.

## 사용법

- `/session-handoff`: 작업 유형 자동 식별 후 인계 문서 작성
- `/session-handoff [topic]`: topic과 유형 hint 지정 (예: `debug-login-lock`, `meta-skill-rework`)

호출은 사용자 명시만 받습니다(`disable-model-invocation: true`). 모델이 임의로 자동 실행하지 않습니다.

## 적용 전제 / 직접 손볼 곳

- **large-context 모델 가정**: 본 대화 전체를 한 번에 흡수합니다. 더 작은 컨텍스트 모델로 새 세션에 진입할 가능성이 있으면 분량 경고가 자동 삽입됩니다.
- **메모리 인덱스(채널 C)는 선택**: `MEMORY.md`, `CLAUDE.md` 같은 메모리 인덱스를 유지하면 자동 grep, 없으면 본 대화 read + 사용자 명시 2채널로 동작합니다.
- **참조 자산 매트릭스는 예시**: SKILL.md의 작업 유형별 메모리 분류(예: "테스트, 검증 게이트 관련 메모리")는 작성자 환경 기준 예시입니다. 자신의 메모리와 문서 체계에 맞게 셀을 조정해 쓰세요. 핵심은 "작업 유형마다 어떤 *종류*의 참조를 끌어올지"이지 특정 분류명이 아닙니다.
- **출력 경로** 기본값은 `.claude/handoff/`. 다른 위치를 쓰면 SKILL.md의 "자동 추론 경로" 절을 고치거나 호출 인자로 경로를 직접 지정하세요.
- **클립보드 명령**은 OS에 맞게: macOS `pbcopy` / Linux `xclip`, `wl-copy` / Windows `clip`.
