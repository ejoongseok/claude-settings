# 콘텐츠 추출 규칙 (외부-가시 문서)

> `/html-explain` 산출물이 "빠르게(가독성) 정확하게(정직성)" 읽히도록 하는 내용 가공 규칙. 생성 시 적용하고, `--external` 모드에서는 특히 엄격히 본다.

## 두 축 · 10원칙

| # | 축 | 원칙 | 한 줄 |
|---|----|------|-------|
| 1 | 정직성 | 사실 기반 톤 | 내부 분석·담당자·주관 판단 배제, 간결한 사실 |
| 2 | 가독성 | 내부 경로 금지 | 로컬 도구·설정 파일 경로 참조 금지. 필요하면 내용을 직접 풀어 쓴다 |
| 3 | 가독성 | 저맥락 | 고유 용어·약어는 괄호 부연 또는 일상 표현으로 |
| 4 | 가독성 | Outside-in | 바깥(체감·동기) → 안쪽(구현) 순. 첫 단락에 기술 용어 금지 |
| 5 | 가독성 | 스캔 > 정독 | 표 > 라벨 불릿 > 산문. 3개 이상 속성은 표로 |
| 6 | 정직성 | What > How | 주어에 수단(서버·프로토콜)이 아니라 본질 변화를 둔다 |
| 7 | 정직성 | 단계 효과는 그 시점 기준 | 최종 상태 효과를 중간 단계 효과처럼 쓰지 않는다 |
| 8 | 가독성 | 내부 상수·플래그 금지 | 코드 상수·내부 플래그명 복붙 금지 |
| 9 | 정직성 | 정량·절대 표기 | "최근/곧/대략" 대신 구체 날짜·숫자·버전 |
| 10 | 정직성 | 추측·단정 구분 | 확정은 근거와 함께, 추정은 `추정`/`TBD` 표시 |

## 리트머스

- **원칙 6**: 문장에서 수단 단어(서버명·프로토콜·엔드포인트)를 지워도 의미가 남는가? 남아야 한다.
- **원칙 10**: 각 주장에 근거를 물었을 때 구체 출처(측정값·직접 인용 등)로 답할 수 있는가? 없으면 `추정` 표시.
