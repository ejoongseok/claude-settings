# /html-explain

긴 문서나 대화를 **핵심부터 한눈에 들어오는 인터랙티브 HTML 한 장**으로 바꿔 줍니다. 원본은 그대로 두고 별도 `.html`만 만듭니다.

## 왜 쓰나

길고 딱딱한 문서·긴 대화는 정작 중요한 게 묻혀서, 읽는 사람이 매번 "그래서 결론이 뭔데?"를 스스로 파내야 합니다. 이 스킬은 그 부담을 **읽는 사람에게서 도구로** 옮깁니다 — 내용을 결론부터, 훑기만 해도 이해되게 다시 짜서 보여줍니다.

## 쓰면 얻는 것

- **결론이 맨 위(BLUF)** — 스크롤하기 전에 "무엇을 알아야 하나"가 먼저 잡힙니다.
- **읽지 않고 훑어도 이해** — 긴 산문 대신 표·흐름도·콜아웃으로 구조화합니다.
- **원문 ↔ 쉬운 설명 토글** — 해석을 믿되 원문으로 바로 대조·검증할 수 있습니다.
- **파일 하나로 공유** — 단일 `.html`, 외부 의존 거의 없음(웹폰트 1개). 그대로 열고 그대로 보냅니다.
- **원본은 안전** — 입력 문서를 절대 수정하지 않습니다.

## 무엇을 얻으려는 건가 (목적)

"이해시키는 비용"을 낮추는 것입니다. 같은 내용이라도 전달이 빠르고 정확하면, 읽는 사람의 시간과 오해가 줄어듭니다. 이 스킬의 목적은 예쁜 페이지가 아니라 **빨리·정확히 이해되는 한 장**입니다.

## 이럴 때

회의록 · 진행현황 · 요약 · 제안(RFP) · API 문서 · 외부 공유본을 "빨리, 정확히" 전달하고 싶을 때. 또는 긴 작업 세션을 한 장 요약으로 남기고 싶을 때.

## 이런 건 아닙니다

단순 Markdown→HTML 변환기가 아닙니다. 포맷만 바꾸는 게 아니라 **이해되게 재구성**하는 것이 핵심입니다. 화려한 대시보드도 아니고, 읽기에 최적인 담백한 문서를 지향합니다.

## 설치

이 폴더를 `~/.claude/skills/` 또는 프로젝트 `.claude/skills/` 아래에 그대로 두면 됩니다. 외부 파일 의존이 없습니다(웹폰트 CDN 1개 제외 — 없으면 시스템 폰트로 대체).

## 사용법

- `/html-explain path/to/doc.md` — 그 문서를 HTML로
- `/html-explain` — 지금까지의 세션 대화를 요약해 HTML로
- `/html-explain 주제힌트 --kind=summary` — 주제 힌트 + 종류 지정
- `/html-explain proposal.md --external` — 외부 공유용(정확성 검토 게이트 포함)

## 산출물

작업 폴더의 `rendered/{날짜}-{제목}.html`. 단일 파일이라 그대로 열거나 공유할 수 있습니다(폰트만 인터넷 필요, 없으면 시스템 폰트).

## 종류 (kind — 자동 판별, 생략 가능)

회의록 · 진행현황 · 요약 · RFP/제안 · API · 외부 제공용

## 품질 기준

디자인·문서 품질은 임의 감각이 아니라 **확립된 공개 원칙**에 근거합니다 — Nielsen 사용성 휴리스틱, Norman 인지 모델, Dieter Rams, Gestalt, Laws of UX, WCAG, Diátaxis 등 (`assets/design-principles.md`).

## 참고

- 외부 공유 전에는 "원문" 토글로 정확성을 확인하세요(요약은 해석 레이어입니다).
- 쓸수록 좋아집니다 — 결과에 피드백을 주면 케이스 학습(`assets/playbook.md`)에 쌓입니다.
