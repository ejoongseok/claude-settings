> **외부-가시 문서**: [`rules/external-doc.md`](../../rules/external-doc.md) 10원칙(가독성 5 + 정직성 5) 준수. 이 스킬 산출물 자체가 외부 독자 대상이며, Phase 3 재구성과 Phase 4 자기 검증은 10원칙 전항을 그대로 체크리스트로 돌린다. 원칙 해설과 예시는 `external-doc.md` 참조, 재작성 금지(CONTRACT 10절).

## 역할

내부 기록, 분석, 초안으로 작성된 마크다운 문서를, 작업 컨텍스트가 없는 제3자(동료 개발자, 의사결정자, 이해관계자, 고객)가 **빠르게(가독성) 정확하게(정직성)** 이해할 수 있는 버전으로 **새 파일로 생성**한다.

## 핵심 원칙

- **원본 불변**: 원본 파일은 Read만. 절대 Edit/Write 하지 않는다.
- **사실 보존**: 원문에 없는 수치, 날짜, 단정, 담당자, 경로를 추가하지 않는다. 구조 재배치, 저맥락화, 톤 가공만 허용.
- **유출 방어**: 내부 식별자(이메일, 사번, IP, API 키 흔적)는 민감 정보 스캔으로 감지 후 사용자 확인 없이 외부 버전에 실리지 않도록 한다.
- **결정은 사용자**: 원본의 카테고리(daily/memo/learn 등) 판정을 기준으로 실행을 거부하지 않는다. 힌트만 제공하고 계속 진행한다.

## 다른 스킬과의 경계

| 질문 | 담당 | 이 스킬에서 |
|------|------|-----------|
| 수신자별 메시지 가공 (슬랙, 이메일, 보고서) | `/draft` | 다루지 않음 |
| 비-MD 문서(PDF, PPTX, XLSX) 전처리 | `/parse-doc` | 다루지 않음 |
| PR 본문 생성 | `/pr` | 다루지 않음 |
| 이슈 본문 생성 | `/issue` | 다루지 않음 |
| ADR/PRD/SRS 초안 작성 | `/adr`, `/prd`, `/srs` | 다루지 않음 |
| **기존 MD 문서를 외부용으로 리라이팅(새 파일)** | **이 스킬** | 핵심 |

## 외부 데이터 의존

| 데이터 | 경로 | 필수/선택 | 부재 시 동작 |
|--------|------|:------:|------------|
| 프로젝트 컨텍스트 (회사 도메인, 고유 용어 힌트) | `CLAUDE.md` | 선택 | 프로젝트 고유 용어 힌트 없이 진행, `[프로젝트 규칙 미확인]` 태그 |
| 비즈니스 규칙 (도메인 약어, 내부 라벨 힌트) | `.local.claude/biz-rules.md` | 선택 | 저맥락화는 일반 기준만 적용 |
| 팀원 매핑 (실명 판정) | `.local.claude/team.md` | 선택 | 실명 연속 탐지를 휴리스틱(한글 2~4자 연속 3인 이상)으로 대체 |
| 인프라 정보 (사내 시스템 URL 힌트) | `.local.claude/INFRASTRUCTURE.md` | 선택 | 일반 내부 URL 패턴(`*.local`, `10.*` 등)만 스캔 |

## 입력 처리

### `$ARGUMENTS`

| 입력 | 해석 |
|------|------|
| `/rewrite-external <파일경로>` | 단건 모드. 해당 MD 파일만 리라이팅 |
| `/rewrite-external <디렉터리경로>` | 일괄 모드. 디렉터리 내 `**/*.md` 재귀 수집 |
| `/rewrite-external <글롭패턴>` | 일괄 모드. 패턴 매칭 파일 전체 |
| `/rewrite-external` (인자 없음) | 대화 맥락에서 추론. 모호하면 질문 |

경로는 절대, 상대, Windows, POSIX 형태 모두 허용. 공백 포함 경로는 따옴표 권장.

### 입력 유효성 검사 (Phase 1 진입 전)

| 상황 | 처리 |
|------|------|
| 경로가 존재하지 않음 | 에러 메시지 + 유사 경로 제안 (Glob으로 근접 매칭 3개) |
| 파일이 `.md`/`.markdown`이 아님 | 거절 + `/parse-doc` 사용 안내 (비-MD 문서는 먼저 파싱 필요) |
| 빈 파일 / 본문 20자 미만 | 거절. 리라이팅할 내용 부재 |
| 바이너리 파일(MIME 감지 실패) | 거절 |
| 읽기 권한 없음 | 에러 표시, 다음 파일로 skip (일괄 모드) |
| 원본 크기 상한 | 3000줄 또는 200KB 초과 시 경고 + 사용자 승인 후 진행 (Read 반복과 컨텍스트 부담 고지) |

## 입력 모드

### 단건 모드

- 입력이 `.md` 단일 파일 경로일 때
- 출력: `.local.claude/external/{YYYY-MM-DD}-{원본파일명 확장자 제거}.md`
- 예: `/rewrite-external .local.claude/projects/{project}/README.md` 는 `.local.claude/external/{YYYY-MM-DD}-README.md` 로 생성
- **상대 링크 경고**: 원본이 같은 폴더의 다른 MD를 상대 경로로 참조하면(`./notes.md`, `../overview.md` 등) 단건 모드에서는 해당 경로가 깨짐. 사용자에게 경고 + 각 링크에 대해 3가지 옵션 제시
  - (1) 일괄 모드로 전환 권유 (디렉터리 단위 변환)
  - (2) 해당 링크를 **제거**하고 본문에 핵심 내용을 풀어 쓰기
  - (3) 절대 URL로 치환 가능한 경우 치환

### 일괄 모드

- 입력이 디렉터리 또는 글롭 패턴일 때
- **파일 수집 규칙**
  - Glob으로 `**/*.md` 재귀 수집
  - **제외 파일명** (대소문자 무시): `INDEX.md`, `MEMORY.md`, `PLAN.md`, `TEMPLATE.md`, `TODO.md`
  - **제외 경로 조각**: `bot/` 하위 (AI 전용 재구성본)
  - 숨김 폴더(`.` 시작) 하위 제외
- **출력 구조**: 원본 디렉터리 트리를 복제
  - 입력 루트명을 `.local.claude/external/{입력루트슬러그}/` 로 매핑
  - **슬러그 규칙**: 입력 루트 디렉터리의 basename만 사용. 소문자화, 공백과 언더스코어는 하이픈으로 치환, 영숫자와 하이픈 외 문자 제거. 빈 문자열이면 `YYYY-MM-DD-rewrite` 로 대체
  - 원본 상대 경로를 유지하여 상호 참조 링크 자동 복원
  - 예: 입력 `.local.claude/projects/{project}/` 내 `overview.md` 는 `.local.claude/external/{project}/overview.md` 로 생성
- **사전 확인**: 대상 파일이 5개를 초과하면 목록을 사용자에게 먼저 표시하고 진행 승인을 받은 뒤 쓰기 시작한다.
- **덮어쓰기**: 출력 파일이 이미 존재하면 기본 skip + 사용자에게 재작성 여부를 묻는다.
- **실패 격리**: 파일별로 Phase 1~5를 독립 실행. 중간 실패는 해당 파일만 skip하고 계속 진행.

## 링크, URL, 이미지 정책

| 유형 | 처리 |
|------|------|
| **내부 문서 경로** (`.claude/*`, `.local.claude/*`, `CLAUDE.md` 등) | **제거**. 필요하면 내용을 본문에 풀어 쓰기 |
| **외부 URL** (`https://`, `http://`, `mailto:` 등) | **유지**. 다만 내부 시스템 URL(사내 Jenkins, Grafana, NAS 등)은 마스킹 또는 제거 |
| **상대 MD 링크** (`./foo.md`, `../bar.md`) | 단건 모드: 경고 후 사용자 선택(제거, 풀어쓰기, 절대 URL 치환). 일괄 모드: 트리 복제로 자동 복원 |
| **앵커 링크** (`#section-name`) | 같은 문서 내 참조는 유지. 외부 문서 앵커는 대상 유형의 규칙(내부 문서 경로는 제거, 외부 URL은 유지) 적용 |
| **이미지 `![](...)`** | '이미지 처리' 절의 규칙 적용 |

### 이미지 처리

| 원본 링크 형태 | 처리 |
|--------------|------|
| 외부 URL (`https://...`) | 유지 |
| 상대 경로 (`./img/foo.png`) | 단건 모드: `<!-- 이미지 생략: {원본 경로} -->` 주석으로 치환 + 본문에 이미지 설명 한 줄 삽입 / 일괄 모드: 트리 복제 시 상대 경로 그대로 유지하되 이미지 파일 자체는 **복사하지 않음**. 사용자에게 별도 복사 여부를 확인 |
| 내부 경로(`.local.claude/...`) | 제거 + 이미지 설명을 본문에 텍스트로 치환 |

## 프로세스 (5 Phase)

### Phase 1: 원본 이해 + 프로젝트 힌트 로드

- Read로 원본 로드. 2000줄 초과 시 `offset`/`limit`로 분할 로드 후 전체 구조 파악
- 프론트매터 존재 여부, 카테고리, 작성일 파악
- 본문의 인용, 링크, 이미지 경로 목록화 (복원과 치환 대상 체크)
- **프로젝트 특화 힌트 로드** (Phase 3.5 민감 정보 스캔과 Phase 3 저맥락화 정확도를 결정). 단일 메시지 병렬 호출:
  - `CLAUDE.md`: 회사명, 회사 도메인, 제품명 추출
  - `.local.claude/biz-rules.md`: 도메인 약어, 내부 라벨 추출
  - `.local.claude/INFRASTRUCTURE.md`: 사내 시스템 URL 추출
  - `.local.claude/team.md`: 팀원 실명 목록 추출

부재한 파일은 skip (graceful degrade). 로드된 힌트는 Phase 3.5 사내 도메인/실명 패턴과 Phase 3 저맥락화 치환 대상에 추가.

**"이미 외부용 성격" 힌트 체크** (진행 차단 없이 안내만)

원본에 다음 패턴이 **전혀 없으면** 이미 외부용에 가까울 수 있음. 사용자에게 한 줄 안내 후 진행:

- `.claude/`, `.local.claude/`, `CLAUDE.md` 문자열 (Grep)
- 내부 상수와 플래그 패턴 (`[A-Z_]{2,}_YN`, `DEFAULT_[A-Z_]+`, `[A-Z_]{2,}_CD`)
- 주관적 내부 판단 표현 (내부 분석성 라벨, 담당자 주관 추정. 프로젝트별 `biz-rules.md`가 있으면 해당 용어 추가 탐색)

### Phase 2: 본질 추출 (What > How)

- 원문에서 **본질 한 줄** 추출
  - 리트머스: "수단 단어(서버명, 프로토콜명, 라이브러리명, 엔드포인트)를 지워도 의미가 남는가?"
- **보존 목록** 작성 (메모리상 관리, 리라이팅 중 소실 방지)
  - 날짜, 버전, 수치
  - 상태값, 코드값, PR/이슈 번호
  - 표, 경로, 상태 전이 같은 구조적 사실
  - 근거 출처(`파일:줄`, 측정값, 직접 인용)

### Phase 3: 2축 체크로 재구성

작성자 모드로 초안 생성. **원문 사실만 사용, 새 정보 도입 금지**.

`rules/external-doc.md` **10원칙 전항을 체크리스트로 돌린다** (가독성 5: 내부 경로 제거, 저맥락화, Outside-in, 스캔 가능성, 내부 상수 금지 / 정직성 5: 외부 공개 톤, What>How, 단계별 효과, 정량 표기, 추측 표시). 원칙 해설은 원문서 참조, 여기서 재작성하지 않는다.

이 스킬에서의 실행 포인트만 3가지:

- **Outside-in 리드 교체**: 첫 단락은 "사용자 체감, 비즈니스 동기, 팀 관점 한 줄 요약" 순. 원본이 구현 세부로 시작하면 리드를 통째로 교체
- **저맥락화는 괄호 부연 > 삭제**: 제품 고유 용어와 패턴명은 먼저 괄호 한 줄 부연을 시도하고, 맥락 손실 없이 뺄 수 있을 때만 삭제
- **기술 세부는 `<details>` 접기로 보존**: 삭제가 아니라 배치 뒤로. 원본이 제공한 정보량은 유지

**마커 참고 (CONTRACT 13절)**: 내부 문서에서 색상 이모지로 표시된 심각도와 상태 마커는 원칙적으로 텍스트 마커(`[치명적]`/`[OK]`/`[WARN]` 등 CONTRACT 13절 표 참조)로 옮긴다. 단 산출물 대상이 GitHub PR, 이슈, 슬랙 등 **이모지 UX가 긍정적인 외부 플랫폼**이면 원문의 이모지를 보존하거나 해당 플랫폼 관례를 따른다. `rules/external-doc.md` 10원칙이 우선이며 CONTRACT 13절은 내부 일관성 참고 사항.

### 참고: 변환 예시 (제목과 첫 단락)

**Bad (내부 원문 스타일)**

```markdown
# 알림 발송 신규 큐 도입 (1단계)

기존 `NotificationService.java` 에서 `kafka:notify.v1` 토픽으로 메시지를
발행하도록 변경. `isEnabled=Y` 플래그로 신/구경로 분기하며, 실패 시
`DEFAULT_RETRY_COUNT` 만큼 재시도. (.local.claude/projects/{project}/SRS.md 참조)
```

**Good (외부용 리라이팅)**

```markdown
# 알림 발송 책임 분리 (1단계)

사용자 요청 처리 흐름에 알림 전송이 섞여 있어 장애가 번지던 문제를,
알림을 별도 경로로 분리해 실패해도 본 흐름이 멈추지 않도록 만드는
1단계 변경입니다. 기존 발송 동작은 그대로 유지되며, 문제 발생 시
이전 방식으로 즉시 되돌릴 수 있습니다.
```

**무엇이 바뀌었나**

| 원칙 | Bad에서 Good으로 |
|------|-----------|
| What > How (원칙 6) | 수단 "큐 도입"에서 본질 "책임 분리"로 |
| 저맥락화 (원칙 3) | `NotificationService.java`, `kafka:notify.v1` 을 일상 표현으로 |
| Outside-in (원칙 4) | 구현 세부 먼저에서 사용자 체감과 동기 먼저로 |
| 내부 경로 제거 (원칙 2) | `.local.claude/...` 참조 제거 |
| 내부 상수 노출 (원칙 8) | `isEnabled=Y`, `DEFAULT_RETRY_COUNT` 제거 |
| 안전망 명시 | 없던 안전망을 "이전 방식으로 즉시 되돌릴 수 있습니다"로 추가 |

### Phase 3.5: 민감 정보 스캔

Phase 3 초안에 대해 Grep 기반 패턴 검출. 아래 패턴이 하나라도 남아 있으면 **사용자에게 목록 표시 후 처리 방침 확인 전까지 Phase 5(쓰기) 중단**.

| 분류 | 패턴 (예) | 기본 처리 |
|------|----------|----------|
| 이메일 | `[\w.+-]+@[\w-]+\.[\w.-]+` | 역할명으로 치환 (예: `담당자 이메일 마스킹됨`) |
| 내부 도메인 | 회사와 자회사 도메인 패턴 + 사내 시스템 URL (Phase 1에서 로드한 `CLAUDE.md`, `INFRASTRUCTURE.md` 힌트 + 일반 사내 서비스 키워드 `jenkins`, `grafana`, `nas`, `.internal`, `.local`) | 제거 |
| 사내 IP | `10.*`, `192.168.*`, `172.16~31.*` + 사내 공인 IP 범위(힌트 있을 시) | 제거 |
| 비밀번호/토큰 | `api[_-]?key`, `token`, `password`, `Bearer `, `ENC[...]` 흔적 | **무조건 제거** + 사용자 강한 경고 + Phase 5 중단 |
| 사번과 사내 식별자 | `[A-Z_]{2,}_CD`, 사원번호 패턴 (숫자 6~10자리 단독 식별자), `biz-rules.md`에서 로드한 내부 라벨 | 제거 또는 역할명 치환 |
| 실명 나열 | Phase 1에서 로드한 `team.md` 실명이 3개 이상 연속 (미부재 시 한글 2~4자 연속 3인 이상 휴리스틱) | 역할명 치환 권장 안내 (`PL 1명`, `개발자 2명`) |

- 발견 건 수와 처리 방침을 사용자에게 표 형태로 보고
- 일괄 모드에서 다수 파일에 걸쳐 발견되면 파일별로 요약 + 일괄 적용 여부 질의

### Phase 4: 자기 검증 (검토자 모드 전환)

작성 후 **관점을 전환**하여 1회 재읽기. 무한 정제 루프 금지, 1회 검증 후 발견 사항만 보강.

**리트머스 테스트**

| 항목 | 확인 질문 |
|------|----------|
| Outside-in | 첫 단락만 읽어도 외부인이 "무슨 일이 일어나는지" 2~3줄 안에 이해 가능한가? |
| What>How | 제목/첫 단락에서 수단 단어를 지워도 의미가 남는가? |
| 사실 보존 | 원문에 없던 수치, 날짜, 단정, 담당자가 추가되지 않았는가? |
| 추측 표시 | 원본이 분기형이나 추정이면 외부 버전도 분기형과 추정을 유지하는가? |
| 내부 경로 | `.claude/`, `.local.claude/`, `CLAUDE.md` 문자열이 본문에 남아 있지 않은가? Grep으로 최종 확인 |
| 내부 상수 | 코드 상수와 플래그명(`[A-Z_]{2,}_YN`, `DEFAULT_[A-Z_]+`)이 그대로 남아 있지 않은가? |
| 민감 정보 | Phase 3.5 처리 결과가 본문에 반영됐는가? |

**복원성 체크**

Phase 2 보존 목록의 모든 항목(날짜, 수치, PR, 근거 출처, 핵심 표)이 외부 버전에도 남아 있는가? 빠졌으면 보강.

**성공 판정 정량 임계값** (Phase 5 진입 전 모두 통과 필요)

| 지표 | 임계값 | 측정 방법 |
|------|--------|----------|
| 내부 문서 경로 잔존 | **0건** | Grep `\.claude/`, `\.local\.claude/`, `CLAUDE\.md` |
| 비밀번호/토큰 흔적 | **0건** | Phase 3.5 비밀번호/토큰 분류 최종 0 |
| 내부 상수와 플래그 패턴 | **0건** | Grep `[A-Z_]{2,}_YN`, `DEFAULT_[A-Z_]+`, `[A-Z_]{2,}_CD` |
| 보존 목록 유지율 | **100%** | Phase 2 목록 전 항목 본문 존재 확인 |
| 사실 추가 | **0건** | 원본 Grep 기준. 외부 버전의 수치, 날짜, PR 번호가 모두 원본에 존재 |
| 첫 단락 수단 단어 | **0개** | 서버명, 프로토콜명, 라이브러리명이 첫 단락에 없음 |

하나라도 미달이면 Phase 3로 회귀(한 번만). 2회차에도 미달이면 실패 상태로 사용자에게 보고하고 파일 skip.

### Phase 5: 새 파일 쓰기 + 변환 요약

Write로 출력 경로에 신규 생성. 원본 파일은 절대 수정하지 않는다.

**출력 프론트매터 템플릿**

```yaml
---
created: {YYYY-MM-DD}
category: external
source: {원본 파일 상대 경로}
rewritten_by: rewrite-external
---
```

- `source` 필드로 원본 추적성 확보
- 원본 프론트매터 내용을 본문에 복사하지 않는다 (내부 메타 노출 방지). 필요한 정보(작성일 등)만 본문에 자연스럽게 녹인다.

**변환 요약 리포트** (사용자에게 채팅으로 출력, 파일에는 쓰지 않음)

| 항목 | 값 |
|------|-----|
| 본질 한 줄 | Phase 2에서 추출한 본질 |
| 제거한 내부 경로 | N개 (샘플 1~2개 표시) |
| 풀어쓴 용어 | N개 (샘플 1~2개 표시) |
| 민감 정보 처리 | N건 (분류별 집계) |
| 링크 처리 | 유지 M개 / 제거 L개 / 치환 K개 |
| 이미지 처리 | 유지 M개 / 주석화 L개 |
| 보존 확인 | 원문 수치, 날짜, PR 번호 N개 모두 유지 여부 |
| Outside-in 재배치 | 적용 여부 |
| 분할 처리 | (해당 시) 원본 크기 초과로 섹션 분할 진행됨 |

일괄 모드에서는 파일별 요약 + 전체 집계 2단.

## 일괄 모드 실행 순서

1. 대상 파일 목록 수집 (제외 규칙 적용)
2. 5개 초과면 사용자에게 목록 표시 + 승인 요청
3. 기존 출력 파일 존재 여부 확인 후 충돌 목록 표시, 덮어쓰기 여부 확인
4. 파일별 Phase 1~5 독립 실행 (실패 시 해당 파일만 skip)
5. 종료 시 요약 리포트 출력
   ```
   성공: N건
   skip (기존 존재): M건
   실패: K건 (사유 나열)
   민감 정보 처리 합계: 분류별 집계
   ```

## 저장 경로

| 모드 | 경로 |
|------|------|
| 단건 | `.local.claude/external/{YYYY-MM-DD}-{원본파일명 확장자 제거}.md` |
| 일괄 | `.local.claude/external/{입력루트슬러그}/{원본 상대 경로}` |

디렉터리 부재 시 자동 생성. 기존 파일 존재 시 기본 skip + 사용자에게 재작성 여부 질의.

## 분량 임계

| 대상 | 임계 1 (알림) | 임계 2 (강제 분리) |
|------|:----------:|:--------------:|
| 원본 파일 크기 | 1500줄 | 3000줄 또는 200KB 초과 시 사용자 승인 후 섹션 단위 분할 진행 |
| 일괄 모드 수집 파일 수 | 5개 초과 시 목록 표시와 승인 요청 | 20개 초과 시 배치 분할 권고 (AskUser 3지선다 제시) |

임계 2를 초과해 분할 진행한 경우 변환 요약 리포트에 `[분할 처리]` 태그 표기.

## 다음 스킬 연결

- 생성된 외부용 문서를 기반으로 수신자별 메시지를 만들고 싶으면 `/draft`
- 생성된 외부용 문서를 PR 본문이나 이슈 본문으로 다듬고 싶으면 `/pr` 또는 `/issue`
- 비-MD 원본(PDF, PPTX, XLSX)이 입력으로 들어오면 먼저 `/parse-doc` 으로 MD 변환

## 제약조건

- 원본 파일 Edit/Write 금지 (Read만 허용)
- 원문에 없는 사실, 수치, 날짜, 담당자, PR 번호 추가 금지
- 원문이 추정으로 표시한 내용을 확정 사실처럼 바꾸지 않기
- 내부 문서 경로(`.claude/*`, `.local.claude/*`, `CLAUDE.md`)를 본문, 링크, 근거 어떤 형태로든 포함 금지
- 민감 정보(비밀번호, 토큰) 검출 시 사용자 확인 없이 Phase 5 진입 금지
- 무한 자기 검증 루프 금지. Phase 4는 1회만
- **외부-가시 문서 공통 원칙 준수**: `rules/external-doc.md` (가독성 5 + 정직성 5). 이 스킬 산출물은 본질적으로 외부 독자 대상.

## 검증 시나리오

공통 3블록(빈 / 부분 / 풀 데이터)은 **CONTRACT 6-1절** 참조.

### 이 스킬의 고유 실패 시나리오

**[데이터 결함]** 프론트매터 카테고리가 "내부 전용"(memo, daily, brainstorm, learn)인데 리라이팅 요청
- 신호: 원본 frontmatter `category` 값 또는 경로가 내부 전용 카테고리
- 대응: 변환 요약 리포트 상단에 `[내부 기록 원본 — 외부 리라이팅 품질 저하 가능]` 경고 + 진행 (핵심 원칙: 카테고리 판정을 기준으로 실행 거부하지 않음)

**[데이터 결함]** 원본에 민감 정보(토큰, 비밀번호) 확정 탐지
- 신호: Phase 3.5 패턴 매칭에서 `api[_-]?key`, `Bearer `, `ENC[...]` 등 토큰/비밀번호 분류 1건 이상
- 대응: **Phase 5 쓰기 즉시 중단**, 사용자에게 강한 경고 + 원본 자체 점검 권고 (외부 문서 이전에 원본이 이미 오염된 상태)

**[사용자 개입 필요]** 상대 MD 링크 단건 모드에서 발견
- 신호: 원본이 `./*.md`, `../*.md` 참조 보유
- 대응: 3지선다 확인 (일괄 모드 전환 / 제거 후 풀어쓰기 / 절대 URL 치환)

**[환경/규모]** 원본 크기 3000줄 또는 200KB 초과
- 신호: Read offset/limit 분할 필요
- 대응: 사용자 승인 후 섹션 단위 분할 진행, 변환 요약 리포트에 `[분할 처리]` 표기
