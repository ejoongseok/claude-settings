## 컨텍스트

- 목표(What): 장애와 긴급 문의의 1차 대응 책임을 명시적 로테이션으로 분산한다. 특정 인원 집중과 야간 무대응 해소
- 합의: 주 1교대, 1차와 백업 2인 체제, 기존 도구로 시작. 본 플랜은 파일럿 개시까지 다루고, 2주 파일럿 수행과 회고와 정식 도입 결정은 별도 작업
- 전제(선행 승인, 자원, 인물): 팀 5인 전원 참여 동의 필요(미확보 시 단계 1에서 중단), 팀 리드 승인
- 미해소 질문: 없음

## 실행 단계

### 1. 참여 합의와 대응 범위 정의
- 대상, 주체, 수치: 팀 5인 전원, 리드 주재 30분 미팅 1회. 대응 대상(P1 장애 / P2 문의)과 대응 시간대(평일 09시에서 22시)를 1장 문서로 확정
- 완료 판정: 5인 서면 동의와 대응 범위 문서의 팀 채널 게시

### 2. 로테이션 표 작성과 캘린더 등록
- 대상, 주체, 수치: 리드가 향후 10주치 로테이션 표(1차와 백업)를 입사일 오름차순 순번으로 작성, 공유 캘린더에 주별 일정 등록
- 완료 판정: 10주 표 게시와 캘린더 초대 전원 수락

### 3. 대응 런북 초안
- 대상, 주체, 수치: 장애 등급 판정, 에스컬레이션 경로, 연락처를 담은 1장 런북. 담당은 리드와 자원자 1명(단계 1 미팅에서 확정, 자원자가 없으면 로테이션 순번 1번이 맡는다)
- 완료 판정: 런북 팀 리뷰 1회 통과와 위치 고정(팀 위키)

### 4. 파일럿 개시
- 대상, 주체, 수치: 로테이션 표 1주차부터 2주 시범 운영을 개시. 리드가 시작 공지(대응 범위 문서와 런북 링크 포함)를 팀 채널에 게시
- 완료 판정: 시작 공지 게시와 1주차 담당자 2명(1차, 백업)의 확인 응답
- 비고: 2주 파일럿 수행, 회고(대응 건수, 지연, 부담 5점 설문 집계), 정식 도입 또는 조정 결정은 별도 작업(본 플랜 범위 외)
