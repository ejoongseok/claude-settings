주문 화면 흐름 (CRUD 프로토타입) — 예시

목록·검색·등록·상세·수정·삭제 — 클릭으로 전환

이 화면들·버튼들·흐름이면 되나요?

동작하는 와이어입니다 — 검색, [+ 등록]으로 새로 만들기, 행 클릭으로 상세, 상세에서 [수정]·[삭제], 행에서 바로 삭제, 페이징까지. 실제 클릭해 보며 "등록 버튼이 여기 맞나, 삭제는 어디서 하나, 수정 화면은 어떤 필드인가"를 고객·기획과 맞춥니다. (픽셀 디자인 아닌 흐름·동작 합의용)

합의
이 화면·버튼·흐름이 맞나요? 등록 버튼 위치, 삭제를 상세·행 양쪽에서 다 할지(권한은?), 수정 폼의 필드 구성, 저장 후 목록/상세 어디로 갈지, 삭제 시 확인을 둘지 — 직접 클릭해 화면 요구와 맞춥니다.
참고
저충실 와이어 + 동작(흐름) 합의용. 픽셀 디자인은 별도 UI 디자인 영역. 데이터는 예시(삭제해도 "처음으로"로 복구되진 않음 — 새로고침).
