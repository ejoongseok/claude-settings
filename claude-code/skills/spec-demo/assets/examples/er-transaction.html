주문 트랜잭션 전파 (ER) — 예시

생성·변경·삭제가 관계선을 타고 위아래로 전파된다

상황 — 주문 생성·변경·삭제

주문 한 건을 만들면 헤더와 항목이 함께 생기고, root를 바꾸거나 지우면 하위로 전파됩니다.

  • 생성 — "주문 생성"하면 헤더 1 + 항목 N이 한 트랜잭션으로 동시에 생깁니다(함께 깜빡, 1:N 관계선 펄스).
  • 변경 — 항목 상태를 바꾸면 헤더로 집계(상향), 프로젝트를 취소하면 항목으로 cascade되고 다시 헤더로 재집계(하향+연쇄).
  • 삭제 — root(헤더)를 지우면 항목까지 cascade 삭제됩니다. 상품은 마스터라 전파 대상이 아닙니다.

참고
헤더·항목의 1:N, 프로젝트→항목 전파는 정해진 규칙 기준으로 둡니다. 생성 건수·집계·cascade의 구체 규칙은 가정(합의 대상). 상품은 마스터라 전파 대상 아님.
