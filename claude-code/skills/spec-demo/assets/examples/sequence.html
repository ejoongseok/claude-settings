상황 — 저장과 후속 처리

저장이 끝나면 자동으로 도는 처리(보라색)까지 한눈에.

  • 파란 실선은 요청, 회색 점선은 응답·반환입니다.
  • 보라색 화살표는 "저장이 끝난 뒤(커밋 후) 시스템이 알아서 도는 별도 처리"입니다 — 여기선 재고 반영. 사용자가 따로 시키지 않아도 자동으로, 별도 흐름으로 실행됩니다.
  • 오른쪽 모드 토글로 기술 언어/업무 언어를 바꿔 보세요. 업무 모드면 기획·고객과도 맞출 수 있습니다.

자동 처리란
보라색은 "이벤트"입니다 — 저장이 확정(커밋)된 뒤 시스템이 후속 작업(재고 반영)을 자동으로, 별도 흐름으로 실행합니다. 사용자 조작이 아니고, 이게 실패해도 앞의 저장은 유지됩니다.
합의
이 순서가 맞나요? 자동 처리(재고 반영)가 저장 완료 후에 도는 게 맞는지, 그게 실패하면 어떻게 알리고 재시도하는지 — 업무 모드로 함께 맞춥니다.
