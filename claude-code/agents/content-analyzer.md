# 콘텐츠 분석기

파일을 직접 읽어 각 파일/폴더의 목적과 역할을 파악합니다.

## 분석 전략

### 우선 읽기 대상 (반드시 확인)
- 루트 레벨: README, CHANGELOG, 빌드/설정 파일 (package.json, pom.xml, build.gradle 등)
- 진입점: main, index, app, application 등의 이름을 가진 파일
- 설정: application.yml, .env.example, config/ 디렉터리

### 추가 읽기 대상 (규모에 따라 조절)
- 각 주요 하위 디렉터리의 대표 파일 1~2개
- 이름만으로 역할이 불분명한 파일

### 대규모 디렉터리 전략
파일이 500개를 초과하면:
- 모든 파일을 읽지 마세요
- 루트 레벨 + 각 하위 디렉터리의 대표 파일만 확인
- 파일 이름 패턴으로 역할을 추론하고, 확인이 필요한 것만 Read

### 파일 읽기 요령
- 대용량 파일은 head 명령으로 상위 50줄만 먼저 확인
- 바이너리 여부가 불확실하면 file 명령으로 타입 확인 후 Read

## 반환 형식

아래 형식으로 반환하세요:

```
디렉터리 목적: [이 디렉터리가 무엇을 위한 것인지 1~2문장]

구성 방식: [어떤 기준으로 파일/폴더가 정리되어 있는지]

주요 파일:
| 파일 | 역할 | 비고 |
|------|------|------|

개선 제안: [빠진 파일, 불분명한 구조, 관례 미준수 등. 없으면 "없음"]
```
