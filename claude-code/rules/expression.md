# 표현과 어조 규칙 (산출물 작성 시 자동 로드)

## "박다, 박혀 있다, 박제" 표현 금지

기록, 메모, 코드 설명, 기술 답변 전 범위. 중립어로 대체:

| 금지 | 대체 |
|------|------|
| 문서에 박아두다 | 정리한다 / 기록한다 / 남긴다 |
| 메모리에 박제하다 | 저장한다 / 보관한다 |
| 규칙으로 박다 | 명시한다 / 규정으로 기록한다 |
| 코드에 박혀 있다 | 정의되어 있다 / 하드코딩되어 있다 |
| 값을 박다 | 지정한다 / 설정한다 / 고정한다 |
| 체크리스트에 박아놓다 | 추가한다 / 넣어둔다 |
| 못박아두다 | 명시한다 / 확정한다 / 고정한다 |

- 이유: "박다"는 못이나 말뚝처럼 거칠게 고정한다는 원래 뜻. 문서와 메모 맥락에서 어조가 과해 내용 전달을 방해
- 점검: 답변과 문서 발송 전 `박[다아어혀힌힘]|박제|박힌|박혀|박아` 패턴 자가 스캔. 발견 시 위 표 기준으로 교체. 기술 맥락(하드코딩 설명 등)도 예외 없음

## 비격식 구어체 표현 금지

기술 문서와 답변에서 구어체 동사는 상황을 뭉뚱그린다. 같은 "먹힌다"가 "적용된다"와 "소실됐다" 둘 다로 읽히는 식이라, 무엇이 일어났는지가 드러나는 정확한 동사로 쓴다. 상황별 대체:

| 상황 | 구어체 (금지) | 대체 |
|------|------|------|
| 실행 중단 | 죽었다, 뻗었다 | 중단됐다 / 멈췄다 / 응답하지 않는다 |
| 오류 발생 | 터진다, 터졌다 | 예외가 발생한다 / 오류가 난다 |
| 손상과 유실 | 깨졌다, 날아갔다, 꼬였다 | 실패했다 / 손상됐다 / 유실됐다 / 어긋났다 |
| 급변과 비정상 종료 | 튄다, 튀었다 | 급변한다 / 비정상 종료됐다 |
| 전송과 호출 | 쏜다 | 전송한다 / 호출한다 / 보낸다 |
| 적용과 소실 | 먹힌다, 먹혔다, 씹혔다 | 적용된다 / 반영된다 (부정이면 적용되지 않는다) / 유실됐다 |
| 기록 | 찍는다, 찍었다 | 기록한다 / 출력한다 / 남긴다 |
| 판단 서술 | 말이 안 된다 | 성립하지 않는다 / 앞뒤가 맞지 않는다 / 타당하지 않다 |
| 고정 | 박힌다, 박혀 있다 | 하드코딩되어 있다 / 고정되어 있다 (위 "박다" 표와 같은 계열) |

- **이 표는 대표 예이지 전체 목록이 아니다.** 판단 기준은 "말로는 자연스러워도 격식 문서에 그대로 쓸 수 없는 구어체 동사인가"다. 표에 없어도 해당하면 대상이다 ("값이 널뛴다", "시스템이 굴러간다" 같은 것)
- 예외: 사용자 발화 원문 인용. 인용 밖 설명에서만 대체한다. dead code의 번역어처럼 통용되는 기술 용어("죽은 코드", "죽은 링크")는 구어체가 아니라 용어이므로 대상이 아니다. 문서가 일관되게 쓰는 개념 라벨과 은유 체계("원석"과 "알맹이" 같은 명명)도 그 문서의 용어로 취급해 유지한다. 축을 구분한다: 은유(비유 장치)는 격식 문서에서도 정당하지만, 개념을 담지 않는 캐주얼한 단어 선택("뽑기", "반쪽" 같은 것)은 제목이라도 대체 대상이다
- 점검: 답변과 문서 발송 전 위 기준 자가 스캔. 훅이 대표 패턴을 기계 검출하고, 목록 밖 표현은 판단으로 잡는다

## 이모지 상세 대체 가이드

스킬이 생성하는 모든 문서에서 이모지 사용 금지. 상태와 강조는 텍스트 레이블로 대체:

- `✅ → [완료]/OK`
- `❌ → [실패]/FAIL`
- `⚠️ → [주의]/WARN`
- `🚨 → [중요]/CRITICAL`
- `📌 📝 🔧 ✨ 🎯` → 제거 또는 괄호 레이블

기존 문서의 이모지는 손대지 않음. 앞으로 **새로 생성하는 문서**에만 적용. 일괄 교체는 별도 요청 시.

## 특수문자와 특수 번호기호 대신 자연어 (훅 검출)

프로즈 기준 금지와 대체. 코드, 경로, 식별자, 다이어그램 문법, 일반 문장부호, 상태 기호(✓ ✗ △), 일반 숫자는 예외:

| 금지 | 대체 |
|------|------|
| 화살표 →, 프로즈 속 ->, => | "다음에", "이어서", "~로 이어진다" |
| 나열 가운뎃점 (A·B·C) | 쉼표나 "와", "그리고" |
| 문장을 잇는 긴 줄표(em dash) | 문장 분리, 괄호, 쉼표 |
| 정의용 등호 ("제이=WMS", "A = B") | "제이가 WMS를 담당"처럼 서술 |
| 별표 강조 ★, 삼각 불릿 ▶ ▸, 참고표 ※, 섹션기호 § | "특히", "핵심은" 같은 말, 일반 목록 |
| 원문자 ①②③, 괄호 숫자 ⑴, 원문자 한글 ㉠, 로마숫자 Ⅰ Ⅱ | 일반 숫자 목록 (1. 2. 3.) |
| 별점 척도 ★★★☆☆ | 숫자 척도 (3/5) |

- 산식이 필요하면 백틱으로 감싸 코드로 취급
- 기존 문서의 기호는 그 문서를 손댈 때만 정리

## 한자 대신 한글 (훅 검출)

- 한국어 문장에 한자를 섞지 않는다: 効率 대신 효율, 確認 대신 확인, 對備 대신 대비
- 예외: 고유명사, 코드와 식별자, 원문 인용

## 마크다운 블록 빈 줄 (표 앞은 훅 검출)

- 표 앞에 빈 줄 필수. 없으면 표가 텍스트 덩어리로 렌더링됨
- blockquote(>) 여러 줄은 빈 줄로 분리해야 줄별로 렌더링됨
- 불릿 리스트와 코드 블록도 앞뒤 빈 줄 권장

## 프로젝트 단위 제외

- 저장소 상위 경로에 `.style-lint-off` 파일이 있으면 훅이 그 저장소 전체를 건너뛴다
- 용도: 원문 문체 보존이 우선인 저장소(코퍼스, 페르소나), 외부에서 수령한 저장소
- 검출 스크립트: `~/.claude/hooks/style_lint.py` (이 리포 claude-code/hooks/style_lint.py를 설치한 것. 경고 모드, 새로 쓰인 텍스트만 검사, 기존 내용 소급 없음)
